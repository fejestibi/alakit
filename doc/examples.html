Alakit Gyakorlati Példák

Tanuld meg az Alakit használatát valódi kódpéldákon keresztül. A "Hello World"-től az állapotkezelésig mindenre találsz itt választ.

Hogyan épül fel egy Alakit projekt?

Mielőtt belevágnál a kódolásba, fontos megérteni a javasolt fájlszerkezetet. Az Alakit szereti a tisztán szétválasztott HTML (UI) és Rust (Logika) állományokat.

Ajánlott Könyvtárszerkezet

napsugar_app/
├── 📁 src/
│   ├── 📁 controllers/    <-- Ide kerülnek a Rust kontrollerek
│   │   └── greet.rs       <-- Pl. a GreetController (nem kell mod.rs!)
│   └── main.rs            <-- Az alkalmazás belépési pontja
├── 📁 ui/                 <-- Minden frontend fájl (HTML, CSS, Képek)
│   ├── index.html         <-- A fő UI oldal
│   └── style.css
└── Cargo.toml             <-- Rust függőségek

Ez a szerkezet lehetővé teszi, hogy a projekt növekedésével is átlátható maradjon a kód, és az alakit_assets! makró könnyen be tudja ágyazni titkosítva a teljes ui/ mappát a végleges binárisba.

1. Hello World - Interaktív Köszöntés

Ebben a példában egy olyan alkalmazást készítünk, ahol a felhasználó beírja a nevét, rákattint egy gombra, és a Rust backend egy Toast értesítéssel köszönti őt.

1 A Felület tervezése (UI)

A head részben regisztráljuk a kontrollert, a body-ban pedig egy azonosítóval (ID) ellátott konténert használunk az adatok összegyűjtéséhez.

index.html
<alakit-meta controllers="greet" />

<div class="card" id="input-container">
    <h1>Üdvözlünk!</h1>
    <input type="text" name="user_name" placeholder="Hogy hívnak?" />
    
    <!-- A gomb az 'input-container' ID-jú div-ből gyűjti össze az adatokat -->
    <button alakit-cmd="greet:hello" alakit-form="input-container">
        Köszönj!
    </button>
</div>

2 A Logika megírása (Rust)

A kontrollereink a handle metódusban kapják meg a parancsokat. Mivel alakit-form-ot használtunk, az args paraméterben egy JSON string érkezik.

src/controllers/greet.rs
use alakit::{AlakitController, AppContext, alakit_controller};
use serde_json::Value;

#[alakit_controller("greet")]
#[derive(Default)]
pub struct GreetController;

#[async_trait::async_trait]
impl AlakitController for GreetController {
    async fn handle(&self, command: &str, args: &str, ctx: AppContext) {
        if command == "hello" {
            // Adatok kinyerése a JSON stringből
            if let Ok(data) = serde_json::from_str::<Value>(args) {
                let name = data["user_name"].as_str().unwrap_or("Barátom");
                
                // Közvetlen UI manipuláció: Toast értesítés
                ctx.dom.toast_success(&format!("Szia {}, üdv az Alakit világában!", name));
            }
        }
    }
}

2. Counter App - Valódi Állapotkezelés

Ez a példa a projektünkben is megtalálható Counter Demo pontos kódja. Bemutatja a reaktív Store-t: a Rust-ban módosított érték azonnal, kód írása nélkül lefrissül a HTML-ben.

1 UI és Reaktív Kötések

Az alakit-bind attribútum "összeláncolja" a HTML elemet a Store egy kulcsával.

ui/index.html
<!-- Binding a 'count' kulcshoz -->
<div class="value" alakit-bind="count">0</div>

<div class="controls">
    <button alakit-cmd="counter:decrement" class="btn">-</button>
    <button alakit-cmd="counter:reset" class="btn secondary">Reset</button>
    <button alakit-cmd="counter:increment" class="btn">+</button>
</div>

2 A Kontroller Működése

Figyeld meg, hogy a Rust kód csak a Logikát és a Store-t kezeli. Nem kell megkeresnie a DOM elemeket a frissítéshez!

src/controllers/counter.rs
#[async_trait::async_trait]
impl AlakitController for CounterController {
    async fn handle(&self, command: &str, _args: &str, ctx: AppContext) {
        // 1. Jelenlegi állapot lekérése a titkosított Store-ból
        let current_count: i32 = ctx.store.get("count")
            .and_then(|v| v.parse().ok())
            .unwrap_or(0);

        // 2. Logikai döntés
        match command {
            "increment" => {
                ctx.store.set("count", &(current_count + 1).to_string());
                ctx.dom.log(&format!("Logs: {}", current_count + 1));
            },
            "decrement" => {
                ctx.store.set("count", &(current_count - 1).to_string());
            },
            "reset" => {
                ctx.store.set("count", "0");
                ctx.dom.toast_info("Visszaállítva!");
            },
            _ => {}
        }
    }
}

3. Képfeldolgozó App - Natív & Bináris IPC

Ez a példa a legösszetettebb demonstrációnk. Bemutatja az Alakit "No-JS" filozófiájának erejét a natív operációs rendszer hívásokkal, valamint a hibrid megközelítést, amikor hatalmas adatokat (pl. rajzolt vászon) kell a böngészőből a Rust felé továbbítani Bináris IPC csatornán.

1 100% Natív No-JS Megközelítés

Az alapvető képbekéréshez nem használunk HTML <input type="file"> taget és JavaScript olvasót. Helyette a HTML csak egy egyszerű hívást intéz a Rust kontroller felé, ami egy natív Windows/Mac/Linux fájlválasztó ablakot nyit meg, és közvetlenül a memóriába olvassa a képet teljesítményveszteség nélkül.

src/controllers/image_processor.rs (Részlet)
"open" => {
    let file = FileDialog::new()
        .add_filter("Képek", &["png", "jpg", "jpeg"])
        .pick_file();

    if let Some(path) = file {
        match image::open(&path) {
            Ok(img) => {
                // Kép elmentése thread-safe Mutex memóriába feldolgozáshoz
                let mut current = CURRENT_IMAGE.lock().unwrap();
                *current = Some(img.clone());
            }
            Err(_) => { } // Hibakezelés
        }
    }
}

2 Az Ultra-Gyors Bináris IPC (Vízjel funkció)

Amikor a felhasználó a böngészőben rajzol valamit (például egy vízjelet a Canvas-ra), a JavaScript nem tudja JSON string formátumban átküldeni a sok megabájtnyi nyers pixel adatot, mert a böngésző és a háttérfolyamat is belassulna a kódolástól. Erre használjuk a beépített window.alakit_binary() API-t.

ui/app.js (Vízjel beküldése)
// 1. Kiszedjük a RAW RGBA pixeleket a vászonról
const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height);
const pixels = new Uint8Array(imageData.data.buffer);

// 2. Fejléc építése a dimenziókhoz
const headerStr = JSON.stringify({ w: canvas.width, h: canvas.height });
const encoder = new TextEncoder();
const headerBytes = encoder.encode(headerStr);

// 3. Csomag összeállítása: [Hossz 4 bájt] + [Header (UTF-8)] + [Pixelek]
const payload = new Uint8Array(4 + headerBytes.length + pixels.length);
payload[0] = headerBytes.length & 0xFF; // Kis endián kódolás
/* ... adatok betöltése ... */
payload.set(pixels, 4 + headerBytes.length);

// 4. Ultra-Fast kérés a Rust felé "image/watermark" útvonalra
window.alakit_binary(`image/watermark`, payload);

3 A Bináris Adat Fogadása Rust-ban

A Rust oldalon minden kontroller implementálhatja a handle_binary metódust, ami kifejezetten ezeknek az előfeldolgozott bájtoknak a fogadására szolgál. Itt ismét a nagy teljesítményű imageops könyvtár végzi el a munkát az átkapott pixeleken.

src/controllers/image_processor.rs (Részlet)
async fn handle_binary(&self, command: &str, payload: &[u8], ctx: AppContext) {
    if command == "watermark" {
        // ... (Header méret és JSON feldolgozása a fenti payload alapján) ...
        
        // Csomag végén lévő pixel adatok
        let pixels = &payload[4 + header_len..];

        // Memóriában lévő kép lockolása
        let mut current = CURRENT_IMAGE.lock().unwrap();
        if let Some(img) = current.as_mut() {
            if let Some(watermark) = RgbaImage::from_raw(w, h, pixels.to_vec()) {
                let mut processed_img = img.clone();
                
                // Overlay művelet (szupergyors natív ráégetés)
                imageops::overlay(&mut processed_img, &DynamicImage::ImageRgba8(watermark), x, y);

                self.render_image_to_dom(&processed_img, &ctx, "Kész");
                *current = Some(processed_img);
            }
        }
    }
}

3. Hogyan áll össze a projekt?

Az Alakit alkalmazásokhoz két fő dolog kell: a függőségek beállítása és az Engine elindítása.

Cargo.toml
[dependencies]
alakit = { path = "../alakit" }
alakit_macro = { path = "../alakit_macro" }
serde_json = "1.0"
src/main.rs
use alakit_macro::{alakit_assets, alakit_main};

#[alakit_main]
fn main() {
    alakit::AlakitEngine::new("Én Első Appom")
        .embed_ui(alakit_assets!("ui/"))
        .run();
}
