1. Bevezetés

Az Alakit egy hibrid asztali alkalmazás-keretrendszer, amely egyesíti a modern webes technológiák rugalmasságát a Rust biztonságával és teljesítményével. Nem csak egy "újabb Tauri-szerűség", hanem egy merőben új megközelítés a hordozható szoftverfejlesztésben.

Mi az Alakit?

Az Alakit egy köztes utat kínál: használj tiszta HTML/CSS-t a felülethez, de felejtsd el a JavaScript kódolást a logika nagy részéhez.

Alapelvek

  • No-JS (Optionális): Az üzleti logika 90%-a megírható Rust-ban anélkül, hogy egy sor JavaScriptet kellene írnod.
  • Encrypted Backend tároló Az érzékeny adatokat AES-256 alapú titkosítás, memóriazárolás (mlock) és automatikus memóriatörlés (zeroize) védi a Rust oldali RAM-ban (a WebView-ban az adat tiszta szöveg).
  • Auto-discovery: A keretrendszer automatikusan megtalálja és regisztrálja a kontrollereidet a makróknak köszönhetően.

2. Gyorsstart (The 5-minute Setup)

Kezdjük el az Alakit használatát! Egy Workspace alapú struktúrát javasolunk a tiszta szétválasztás érdekében.

Telepítés

A Cargo.toml fájlodba add hozzá a szükséges függőségeket:

[dependencies]
alakit = { path = "../alakit" }
alakit_macro = { path = "../alakit_macro" }

Hello World

Hozzunk létre egy egyszerű gombot, ami Rust oldalon változtat meg egy szöveget:

// main.rs
    use alakit::AlakitEngine;
    use alakit_macro::{alakit_main, alakit_assets};
    
    #[alakit_main]
    fn main() {
        AlakitEngine::new("Hello Alakit")
            .embed_ui(alakit_assets!("ui/"))
            .run();
    }

3. Alapfogalmak

Az Alakit motorja egy egyedi üzenetküldési hídra (The Bridge) épül.

A Híd (IPC)

Amikor egy HTML elemre rákattintasz, az alakit-cmd értékét a rendszer egy IPC (Inter-Process Communication) üzenetté alakítja. Ez a controller:command|args formátumot követi, és azonnal a Rust-os handle metódusba érkezik.

Bináris IPC (Képfeldolgozáshoz és Nagy adatokhoz)

Az Alakit nemcsak egyszerű szövegeket tud küldeni, hanem hatalmas adatmennyiségeket (pl. képek pixeleit, fájlokat) is villámgyorsan. Míg a hagyományos JSON konverzió lelassítaná a böngészőt, a beépített window.alakit_binary() függvény Base64+MessagePack alapokon dekapituláció nélkül továbbítja az adatot a Rust motorba.

Hogyan használd? 

// JS Oldalon
const keppixelek = new Uint8Array([255, 0, 128, ...]);
window.alakit_binary("kep_vezérlo/feldolgoz", keppixelek);

// Rust Oldalon
async fn handle_binary(&self, cmd: &str, payload: &[u8], ctx: AppContext) {
    println!("Kaptam {} bájtot!", payload.len());
}

Namespace-ek

Tipp: Szervezd a kontrollereidet névterek szerint (pl. auth, vault, settings). Egy HTML oldalon több névtér is élhet egyszerre.

Életciklus

1. Startup: Az AlakitEngine inicializálja az eseményhurkot és a Webview-t.
2. Discovery: A Rust regiszter (inventory) feltérképezi a #[alakit_controller] makróval jelölt struct-okat.
3. Runtime: A Store eseményeket küld, a DOM pedig frissíti magát.

4. Frontend: A Deklaratív HTML

Az Alakit HTML nyelve JS-mentes interakciókat tesz lehetővé speciális attribútumokon keresztül.

Részletes Attribútum Referencia

alakit-cmd="namespace:command"

Meghatározza, hogy melyik Rust kontrollernek és melyik parancsnak küldjük az üzenetet.

alakit-event="esemény[.módosító]"

Meghatározza, hogy melyik DOM esemény váltsa ki az IPC üzenetet. Alapértelmezett: click.

Típus / Módosító Példa Leírás
click alakit-event="click" Alapértelmezett. Kattintáskor (vagy érintéskor) fut le.
keyup.enter alakit-event="keyup.enter" Csak az Enter billentyű felengedésekor fut le.
keyup.esc alakit-event="keyup.esc" Csak az Escape gomb megnyomásakor fut le.
input alakit-event="input" Minden egyes karakter beírásakor (vagy törlésekor) lefut.
change alakit-event="change" Csak akkor fut le, ha az elem elveszíti a fókuszt és az értéke megváltozott.
mouseover alakit-event="mouseover" Amikor az egér az elem fölé ér.

alakit-form="container_id"

Ha meg van adva, az Alakit összegyűjti az összes input/select/textarea adatot a megadott ID-jú konténeren belülről, és JSON formátumban küldi el a Rust oldalnak.

  • Használat: Tökéletes összetett űrlapokhoz vagy keresőkhöz.

alakit-args="statikus_adat"

Statikus szöveges paraméter átadása a parancsnak. Ha az elem egy input és nincs alakit-args, az Alakit automatikusan az elem aktuális értékét (value) küldi el.

alakit-bind="key"

Kétirányú kötés. Ha a Rust Store-ban megváltozik a key értéke, az összes ilyen attribútummal rendelkező elem frissül.

alakit-html

Módosító az alakit-bind mellé. Ha jelen van, az Alakit nyers HTML-ként (innerHTML) rendereli az adatot a biztonságos szöveg (innerText) helyett.

alakit-show / alakit-hide

Feltételes megjelenítés. Az elem láthatósága a Store értékének "truthy" (igaz) állapotától függ.

Kézi JavaScript Hívás

Ha egyedi JS logikából szeretnél üzenni a Rust-nak, használd a globális függvényt:

alakit("namespace:command|argumentum");

Ez közvetlenül az IPC csatornára küldi az üzenetet.

5. Backend: Rust Kontrollerek

Itt történik az igazi munka. A kontrollerek felelősek a kérések feldolgozásáért.

#[alakit_controller]

Minden kontrollernek implementálnia kell az AlakitController traitet és használnia kell a regisztrációs makrót:

#[alakit_controller("vault")]
#[derive(Default)]
pub struct VaultController;

#[async_trait::async_trait]
impl AlakitController for VaultController {
    async fn handle(&self, command: &str, args: &str, ctx: AppContext) {
        match command {
            "unlock" => { /* Logika itt */ },
            _ => println!("Ismeretlen parancs"),
        }
    }
}

Automata Felfedezés (Auto-Discovery) Zero-Boilerplate

Nagyobb projekteknél, ahol tucatnyi kontroller van, unalmas lehet minden fájlt kézzel beírni a mod.rs-be. Erre találtuk ki az #[alakit_main] makrót!

Hogyan működik?

Ha ezt ráteszed a main függvényre, a makró teljesen transzparensen beolvassa a src/controllers könyvtárat, beemeli az összes fájlt modulként és lefuttatja a teljes regisztrációt a háttérben. Se build.rs-re, se mod.rs-re nem lesz soha többé szükséged!

6. Adatkezelés: Az Encrypted Store

Ez az Alakit lelke. Egy központi, titkosított memóriatároló, ami szinkronban tartja a Rust-ot és a UI-t.

Biztonság és Memóriavédelem

A Store nem sima String-eket tárol, hanem minden értéket AES-256-GCM algoritmussal titkosít a memóriában. Az Alakit kiemelt figyelmet fordít a RAM-szintű védelemre:

  • Mlock (Memory Locking): A titkosító kulcsok le vannak zárva a fizikai memóriában (region 3.0), így az operációs rendszer nem tudja azokat a swap fájlba (lemezre) írni.
  • Zeroize: Minden érzékeny adat (kulcsok, ideiglenes pufferek, dekódolt értékek) automatikusan felülíródik nullákkal a memóriában, amint a használatuk befejeződött.
⚠️ FONTOS BIZTONSÁGI FIGYELMEZTETÉS:

A fenti védelmi mechanizmusok (titkosítás, mlock, zeroize) kizárólag a Rust oldali tárolásra vonatkoznak. Amikor egy adatot kiküldünk a frontendnek (Webview), az ott plaintext (tiszta szöveg) formában jelenik meg a DOM-ban. Ezért kritikus, hogy ne küldj ki olyan adatot a felületre, aminek ott semmi keresnivalója, mert a Webview-n belül futó scriptek hozzáférhetnek.

Biztonságos RAM API Referencia

A Store API-t közvetlenül a ctx.store objektumon keresztül érheted el a Rust kontrollerekben.

Metódus Argumentumok Leírás
set(key, val) &str, &str Titkosítva elmenti az értéket a RAM-ba és azonnal frissíti a UI-t minden kötött (bind) elemen.
get(key) &str Visszaadja a memóriából dekódolt értéket (Option<String>). Ha az érték sérült vagy nem létezik, None.
remove(key) &str Véglegesen törli az adatot a memóriából és értesíti a UI-t az elem alaphelyzetbe állításáról.
Működési mechanizmus: A set híváskor az Alakit aszinkron IPC üzenetet küld a Webview-nak, így a felület frissítése nem várakoztatja meg a Rust üzleti logikát.

7. RustDOM API referencia

A RustDOM API lehetővé teszi a UI közvetlen manipulálását a Rust kontrollerek handle metódusából, anélkül hogy a Store-ra kellene támaszkodnod.

RustDOM (Globális műveletek)

Ezek a metódusok a ctx.dom objektumon érhetőek el.

Metódus Argumentumok Leírás
get_id(id) &str Visszaad egy Element példányt, amivel az adott ID-jú HTML elemet vezérelheted.
toast_success(msg) &str Zöld színű siker-visszajelzés küldése a felületre.
toast_error(msg) &str Piros színű hibaüzenet küldése.
toast_info(msg) &str Kék színű információs értesítés.
log(msg) &str Fejlesztői üzenet küldése a browser konzolra.

Element (Elem-specifikus műveletek)

Ezek a metódusok a ctx.dom.get_id("...") hívással kapott objektumon érhetőek el.

Metódus Argumentumok Leírás
set_text(text) &str Módosítja az elem belső szövegét (biztonságos).
set_html(html) &str Teljesen felülírja az elem belső HTML tartalmát.
set_value(val) &str Módosítja egy input vagy textarea értékét.
add_class(name) &str CSS osztály hozzáadása az elemhez.
remove_class(name) &str CSS osztály eltávolítása.
set_style(p, v) &str, &str Közvetlen inline CSS módosítás (pl. "color", "red").
set_attribute(n, v) &str, &str Tetszőleges HTML attribútum beállítása (pl. "disabled", "true").
remove() - Véglegesen eltávolítja az elemet a DOM-ból.

8. Csomagolás és Terjesztés

Az Alakit célja a Single Binary elosztás: egyetlen .exe fájl, amiben minden benne van.

Asset Encryption

Az Alakit beépített védelmet nyújt a felületednek. Az alakit_assets!("ui/") makró minden fordításidőben egyedi titkosító kulcsot generál, elkódolja a fájljaidat, és csak a kriptográfiai adatokat menti be az .exe fájlba. Így a Hex Editorok és az Unpackerek (vagy bármely átlagos visszafejtő szoftver) nem tudják ellopni a kódodat. Ráadásul az Alakit dedikált "Inkognitó Módot" használ a böngészőmotorban, így menet közben sem írja ki a fájlokat fizikai lemezre (sem az AppData-ba, sem sehova máshova)!

telepítő mellé.

cargo build --release

Windows Védelem

A Wry-alapú alkalmazásoknál fontos az elektronos aláírás (Code Signing), különben a Windows SmartScreen figyelmeztetést dobhat. Fejlesztés alatt ezt a "More Info -> Run Anyway" opcióval kerülheted meg.

9. Hibakeresés és Diagnosztika

Az Alakit egyik legnagyobb előnye a "Remote-to-Terminal" naplózás, ami lehetővé teszi, hogy a JavaScript oldali eseményeket közvetlenül a Rust konzolon lásd.

Konzol Átirányítás

Nem kell megnyitnod a böngésző fejelsztői eszközeit (DevTools), mert az Alakit automatikusan átirányítja a JS naplókat:

  • console.log() → 🌐 [JS LOG]
  • console.warn() → 🟡 [JS WARN]
  • console.error() → 🔴 [JS ERROR]

Globális Hibakeresés

Ha a frontend kódban váratlan hiba történik (pl. elgépelés vagy null reference), a híd azonnal elkapja azt, és a Rust terminálban megjeleníti a pontos hibaüzenetet és a forrásfájl helyét.

Belső rendszer: Az Alakit a alakit:log IPC csatornát használja ezekre az üzenetekre, így azok nem zavarják a te egyedi kontrollered logikáját.
