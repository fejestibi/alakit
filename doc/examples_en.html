Alakit Practical Examples

Learn how to use Alakit through real code examples. From "Hello World" to state management, you'll find everything here.

How is an Alakit project structured?

Before you start coding, it's important to understand the recommended file structure. Alakit likes clearly separated HTML (UI) and Rust (Logic) files.

Recommended Directory Structure

napsugar_app/
├── 📁 src/
│   ├── 📁 controllers/    <-- This is where Rust controllers go
│   │   ├── mod.rs
│   │   └── greet.rs       <-- E.g., the GreetController
│   └── main.rs            <-- Application entry point
├── 📁 ui/                 <-- All frontend files (HTML, CSS, Images)
│   ├── index.html         <-- Main UI page
│   └── style.css
└── Cargo.toml             <-- Rust dependencies

This structure allows your project to remain organized as it grows, and rust-embed can easily include the entire ui/ folder into the final binary.

1. Hello World - Interactive Greeting

In this example, we'll create an application where the user enters their name, clicks a button, and the Rust backend greets them with a Toast notification.

1 Designing the Interface (UI)

In the head section, we register the controller, and in the body, we use a container with an identifier (ID) to gather data.

index.html
<alakit-meta controllers="greet" />

<div class="card" id="input-container">
    <h1>Welcome!</h1>
    <input type="text" name="user_name" placeholder="What's your name?" />
    
    <!-- The button gathers data from the div with ID 'input-container' -->
    <button alakit-cmd="greet:hello" alakit-form="input-container">
        Greet me!
    </button>
</div>

2 Writing the Logic (Rust)

Our controllers receive commands in the handle method. Since we used alakit-form, the args parameter arrives as a JSON string.

src/controllers/greet.rs
use alakit::{AlakitController, AppContext, alakit_controller};
use serde_json::Value;

#[alakit_controller("greet")]
#[derive(Default)]
pub struct GreetController;

#[async_trait::async_trait]
impl AlakitController for GreetController {
    async fn handle(&self, command: &str, args: &str, ctx: AppContext) {
        if command == "hello" {
            // Extracting data from the JSON string
            if let Ok(data) = serde_json::from_str::<Value>(args) {
                let name = data["user_name"].as_str().unwrap_or("Friend");
                
                // Direct UI manipulation: Toast notification
                ctx.dom.toast_success(&format!("Hi {}, welcome to the world of Alakit!", name));
            }
        }
    }
}

2. Counter App - Real State Management

This example is the exact code for the Counter Demo found in our project. it demonstrates the reactive Store: the value modified in Rust updates immediately in the HTML without writing any extra code.

1 UI and Reactive Bindings

The alakit-bind attribute "chains" the HTML element to a key in the Store.

ui/index.html
<!-- Binding to the 'count' key -->
<div class="value" alakit-bind="count">0</div>

<div class="controls">
    <button alakit-cmd="counter:decrement" class="btn">-</button>
    <button alakit-cmd="counter:reset" class="btn secondary">Reset</button>
    <button alakit-cmd="counter:increment" class="btn">+</button>
</div>

2 Controller Operation

Notice that the Rust code only handles Logic and the Store. It doesn't need to find DOM elements to update them!

src/controllers/counter.rs
#[async_trait::async_trait]
impl AlakitController for CounterController {
    async fn handle(&self, command: &str, _args: &str, ctx: AppContext) {
        // 1. Get current state from the encrypted Store
        let current_count: i32 = ctx.store.get("count")
            .and_then(|v| v.parse().ok())
            .unwrap_or(0);

        // 2. Logical decision
        match command {
            "increment" => {
                ctx.store.set("count", &(current_count + 1).to_string());
                ctx.dom.log(&format!("Logs: {}", current_count + 1));
            },
            "decrement" => {
                ctx.store.set("count", &(current_count - 1).to_string());
            },
            "reset" => {
                ctx.store.set("count", "0");
                ctx.dom.toast_info("Reset!");
            },
            _ => {}
        }
    }
}

3. Image Processor App - Native & Binary IPC

This example is our most complex demonstration. It showcases the power of Alakit's "No-JS" philosophy using native operating system calls, as well as the hybrid approach for transferring huge amounts of data (like a drawn canvas) from the browser to Rust via the Binary IPC channel.

1 100% Native No-JS Approach

For essential image loading, we don't use the HTML <input type="file"> tag and a JavaScript reader. Instead, the HTML simply makes a call to the Rust controller, which opens a native Windows/Mac/Linux file picker dialog and directly reads the image into memory without performance loss.

src/controllers/image_processor.rs (Snippet)
"open" => {
    let file = FileDialog::new()
        .add_filter("Images", &["png", "jpg", "jpeg"])
        .pick_file();

    if let Some(path) = file {
        match image::open(&path) {
            Ok(img) => {
                // Save image to thread-safe Mutex memory for processing
                let mut current = CURRENT_IMAGE.lock().unwrap();
                *current = Some(img.clone());
            }
            Err(_) => { } // Error handling
        }
    }
}

2 Ultra-Fast Binary IPC (Watermark feature)

When a user draws something in the browser (for example, a watermark on a Canvas), JavaScript cannot transmit the megabytes of raw pixel data as a JSON string, because encoding it would slow down both the browser and backend. This is where we use the built-in window.alakit_binary() API.

ui/app.js (Sending Watermark)
// 1. Extract RAW RGBA pixels from the canvas
const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height);
const pixels = new Uint8Array(imageData.data.buffer);

// 2. Build header for dimensions
const headerStr = JSON.stringify({ w: canvas.width, h: canvas.height });
const encoder = new TextEncoder();
const headerBytes = encoder.encode(headerStr);

// 3. Assemble packet: [Length 4 bytes] + [Header (UTF-8)] + [Pixels]
const payload = new Uint8Array(4 + headerBytes.length + pixels.length);
payload[0] = headerBytes.length & 0xFF; // Little-endian encoding
/* ... load remaining data ... */
payload.set(pixels, 4 + headerBytes.length);

// 4. Ultra-Fast request to Rust on the "image/watermark" route
window.alakit_binary(`image/watermark`, payload);

3 Receiving Binary Data in Rust

On the Rust side, every controller can implement the handle_binary method, which is specifically designed to receive these pre-processed bytes. Here, the high-performance imageops library takes over to manipulate the received pixels.

src/controllers/image_processor.rs (Snippet)
async fn handle_binary(&self, command: &str, payload: &[u8], ctx: AppContext) {
    if command == "watermark" {
        // ... (Header size and JSON parsing based on the payload above) ...
        
        // Pixel data at the end of the packet
        let pixels = &payload[4 + header_len..];

        // Lock image in memory
        let mut current = CURRENT_IMAGE.lock().unwrap();
        if let Some(img) = current.as_mut() {
            if let Some(watermark) = RgbaImage::from_raw(w, h, pixels.to_vec()) {
                let mut processed_img = img.clone();
                
                // Overlay operation (super fast native blending)
                imageops::overlay(&mut processed_img, &DynamicImage::ImageRgba8(watermark), x, y);

                self.render_image_to_dom(&processed_img, &ctx, "Done");
                *current = Some(processed_img);
            }
        }
    }
}

3. How does the project come together?

For Alakit applications, you mostly need two things: setting up dependencies and starting the Engine.

Cargo.toml
[dependencies]
alakit = { path = "../alakit" }
alakit_macro = { path = "../alakit_macro" }
serde_json = "1.0"
rust-embed = "8.1"
src/main.rs
#[derive(RustEmbed)]
#[folder = "ui/"]
struct Assets;

fn main() {
    alakit::AlakitEngine::new("My First App")
        .with_assets::<Assets>()
        .run();
}
